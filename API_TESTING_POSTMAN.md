# 📋 API Тестирование: Restful-Booker — Комплексная автоматизация E2E сценариев

## 🧪 Обзор проекта.

**Проект:**<br> 
> Автоматизация комплексного E2E сценария бронирования для учебного API `Restful-Booker`.

**Инструмент:**<br>
> Postman (Collection Runner + Newman CLI).

**Цель:**<br>
> Проверка полного жизненного цикла сущности "Бронирование" с валидацией позитивных и негативных кейсов, включая авторизацию, контроль доступа и обработку граничных условий.

## ⚙️ Конфигурация окружения
Использовалось кастомное окружение Postman (`Restful_Booker_Env`) для управления переменными:
- `baseUrl`: https://restful-booker.herokuapp.com
- `token`: Динамическое хранение токена авторизации после логина.
- `bookingId`: Автоматическое сохранение ID созданной сущности для цепочки последующих запросов (GET, PUT, PATCH, DELETE).
- `lastname`: Переменная для тестирования фильтрации.
